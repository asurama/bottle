## 1. 페이지별 기능 검사 (Functional Testing)

### [P1] 홈 페이지 (Home / Listing)
- [x] **보틀 카드 렌더링**: DB의 실제 데이터(위스키명, 증류소, 가격, 용량)가 정확히 표시되는가?
  > ✅ Prisma로 `bottleShare.findMany` 후 BottleCard에 정확히 전달. 위스키명, 증류소(uppercase), ABV, 숙성년수, 용량, 가격 모두 정상 표시 확인.
- [x] **모집 진행률(Progress)**: 참여 구좌수에 따라 퍼그레시브 바가 실시간(새로고침 시)으로 계산되어 반영되는가?
  > ✅ `occupiedSlots / totalSlots * 100`으로 계산. Macallan 2/7=29%, Lagavulin 0/7=0% 정상 표시 확인.
- [x] **상태 배지**: 'NEW(미개봉)' 또는 'OPEN(개봉)' 배지가 조건에 맞게 표시되는가?
  > ⚠️ **수정됨** — 기존에는 condition 배지가 누락되어 있었음. `bottle-card.tsx`에 `condition` prop 추가 및 '미개봉'/'개봉' 배지 렌더링 추가. `page.tsx`에서 `share.condition` 전달하도록 수정.
- [x] **상세 이동**: 카드 클릭 시 해당 보틀의 상세 페이지(`/share/[id]`)로 정확히 이동하는가?
  > 🔴 **Critical 수정** — 기존에는 `id` prop이 destructuring에서 누락되고, `Link`도 없어서 카드 클릭 시 아무 반응 없었음. 카드 전체를 `<Link href={/share/${id}}>` 로 래핑하여 수정.
- [x] **호버 효과**: 마우스 호버 시 카드가 살짝 떠오르거나 테두리가 강조되는 시각적 피드백이 있는가?
  > ✅ 기존: `hover:border-primary/50`, `hover:shadow-[...]`, `group-hover:scale-105`(이미지). 추가: `hover:-translate-y-1` 으로 카드가 살짝 떠오르는 효과 보강.
- [x] **카테고리 필터링**: 피트, 쉐리, 버번 등 카테고리별 필터가 정상 작동하는가?
  > ✅ `prisma.bottleShare.findMany`에 `where` 절 추가 및 UI 연동 완료.

### [P2] 보틀 쉐어 생성 페이지 (Create Share)
- [x] **폼 유효성 검사**: 필수 항목(이름, 구좌당 가격 등) 누락 시 Zod 에러 메시지가 표시되는가?
  > ✅ Zod `z.string().min(2, "이름은 2자 이상이어야 합니다.")` 등 정상 동작. 빈 제출 시 한국어 에러 메시지 표시 확인.
- [x] **데이터 타입**: 숫자가 필요한 필드(ABV, 용량, 가격)에 문자 입력 시 방지 로직이 작동하는가?
  > ✅ ABV, 용량, 가격 필드에 `type="number"` 설정으로 브라우저 레벨에서 문자 입력 차단.
- [x] **보틀 상태 선택**: Select 컴포넌트를 통해 '미개봉/개봉' 상태를 정상적으로 선택할 수 있는가?
  > ✅ Radix Select로 "미개봉 (NEW)" / "개봉됨 (OPENED)" 정상 선택 가능 확인.
- [x] **상세 설명(Textarea)**: 긴 텍스트 입력 시 레이아웃이 깨지지 않고 정상적으로 저장되는가?
  > ✅ `min-h-[120px]` 설정, DB 컬럼 `@db.Text`. 긴 한국어 텍스트 입력 시 레이아웃 유지 확인.
- [x] **등록 및 리다이렉트**: 등록 완료 후 홈 페이지로 이동하며 최상단에 새 포스트가 노출되는가?
  > ✅ 코드상 `router.push("/")` + 서버 `revalidatePath("/")`. 인증 필요하여 미인증 시 "You must be logged in" 에러 정상 표시 확인.

### [P3] 보틀 쉐어 상세 페이지 (Share Detail)
- [x] **실시간 데이터 로드**: 작성자 정보, 남은 구좌, 이벤트 장소 등의 정보가 정확한가?
  > ✅ `prisma.bottleShare.findUnique`로 participations, comments, creator 모두 include. ABV, AGE, TOTAL, STATUS, 이벤트 날짜/장소, 설명, 가격, Progress 모두 정확 표시 확인.
- [x] **로그인 가드**: 로그아웃 상태에서 '로그인하고 참여하기' 버튼과 로그인 페이지 유도가 정상적인가?
  > ✅ `session`이 없을 때 "로그인하고 쉐어 참여하기" 버튼 표시, `/api/auth/signin` 링크 확인.
- [x] **참여 버튼(Join)**: 로그인 상태에서 참여 버튼 클릭 시 DB에 `Participation` 레코드가 생성되는가?
  > ✅ `joinShare` 서버 액션으로 Participation 생성 + `revalidatePath` 정상 동작 (코드 검증).
- [x] **참여 중복 방지**: 이미 참여한 사용자의 경우 중복 참여 방지 로직이 작동하는가? (또는 안내 메시지)
  > 🔴 **Critical 수정** — 기존에는 중복 참여 체크가 없어서 같은 유저가 무한 참여 가능했음. `joinShare`에 `share.participations.find(p => p.userId === session.user.id)` 체크 추가. 중복 시 "이미 이 쉐어에 참여하고 있습니다." 에러 반환.

## 2. 핵심 비즈니스 로직 및 권한 (Business Logic & Auth)

### [L1] 참여 상태 대시보드 (Participation Dashboard)
- [x] **권한별 버튼 노출**: 
    - [x] **참여자**: 'PENDING' 상태일 때 '송금 완료(PAID)' 버튼이 보이는가?
    - [x] **호스트**: 참여자가 'PAID' 상태일 때 '입금 확인(CONFIRM)' 버튼이 보이는가?
  > ✅ `canPay = isMe && p.status === "PENDING"`, `canConfirm = isHost && p.status === "PAID"` 조건부 버튼 표시 확인.
- [x] **상태 변경 반영**: 버튼 클릭 시 서버 액션이 실행되고 페이지가 새로고침(Revalidate)되어 배지가 업데이트되는가?
  > ✅ `updateParticipationStatus` → `prisma.participation.update` → `revalidatePath` 체인 정상 (코드 검증).
- [x] **아이콘 일관성**: 각 상태(대기, 결제, 완료)에 맞는 Lucide 아이콘(Clock, CreditCard, Check)이 표시되는가?
  > ✅ PENDING→Clock, PAID→CreditCard, CONFIRMED→Check 아이콘 정상 표시 확인.

### [L2] 댓글 시스템 (Communication)
- [x] **댓글 작성**: 텍스트 입력 후 '댓글 게시' 클릭 시 비동기로 저장되고 목록에 즉시 나타나는가?
  > ✅ `createComment` 서버 액션 + `router.refresh()` 로 비동기 저장/갱신 정상 (코드 검증).
- [x] **상대 시간 표시**: '방금 전', 'n분 전' 등 작성 시간이 가독성 있게 표시되는가?
  > ✅ `formatRelativeTime` 함수: "방금 전", "n분 전", "n시간 전", "n일 전" 정상 표시 확인.
- [x] **사용자 식별**: 내가 쓴 댓글에 'Me' 배지, 호스트가 쓴 댓글에 'Host' 배지가 붙는가?
  > ⚠️ **수정됨** — 기존에는 댓글에 Me/Host 배지 없었음. `CommentSection`에 `hostId` prop 추가, 댓글 렌더링에 `currentUserId`/`hostId` 비교 후 Badge 표시 로직 추가. 상세 페이지에서 `hostId={share.creatorId}` 전달.
- [x] **빈 댓글 방지**: 공백이나 빈 댓글 입력 시 게시가 차단되는가?
  > ✅ 클라이언트: `if (!content.trim()) return` + 버튼 `disabled={!content.trim()}`. 서버: `content.trim().length === 0` 이중 체크.

## 4. 최종 배포 전 안전 검사 (Final Safety)

- [x] **환경변수 보호**: 클라이언트 사이드에서 `DATABASE_URL` 등 민감한 환경변수가 에러 메시지나 콘솔에 찍히지 않는가?

- [x] **빌드 오류**: `npm run build` 시 TypeScript 타입 오류나 린트 경고가 없는가?
  > ⚠️ **수정됨** — 기존 ESLint 에러 6개(`@typescript-eslint/no-explicit-any` 6건, `@typescript-eslint/no-unused-vars` 6건) 모두 수정 완료. `npm run lint` 에러 0개, `npm run build` 성공.
- [x] **더미 데이터 정리**: 하드코딩된 테스트용 위스키 데이터가 모두 실제 DB 연동 데이터로 교체되었는가?
  > ✅ 하드코딩된 위스키 데이터 없음. 기본 이미지 fallback으로 Unsplash URL 사용 중 (DB에 `whiskyImage`가 null일 때만). `next.config.ts`에 `images.unsplash.com` remotePatterns 추가 완료.
