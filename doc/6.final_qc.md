## 4. 최종 배포 전 안전 검사 (Final Safety)

- [x] **환경변수 보호**: 클라이언트 사이드에서 `DATABASE_URL` 등 민감한 환경변수가 에러 메시지나 콘솔에 찍히지 않는가?
  > ✅ DB 접근은 서버 컴포넌트/서버 액션에서만 수행. 클라이언트 노출 없음.
- [x] **빌드 오류**: `npm run build` 시 TypeScript 타입 오류나 린트 경고가 없는가?
  > ⚠️ **2차 QC에서 수정** — ESLint 에러 11개 + 경고 7개 → 0개로 수정. `any` 타입 → NextAuth 타입 확장(`next-auth.d.ts`) 적용, 미사용 import/변수 제거.
- [x] **더미 데이터 정리**: 하드코딩된 테스트용 위스키 데이터가 모두 실제 DB 연동 데이터로 교체되었는가?
  > ✅ 하드코딩 없음. Unsplash fallback 이미지만 사용.
- [x] **보안 타입 안전성**: `(session?.user as any)` 같은 unsafe 타입 캐스팅이 제거되었는가?
  > ⚠️ **2차 QC에서 수정** — `src/types/next-auth.d.ts` 타입 선언 추가. `auth.ts`, `header.tsx`, `admin/users/page.tsx`, `actions-admin/user.ts` 에서 `any` 캐스팅 제거.
- [x] **CSS 오류**: 잘못된 Tailwind 클래스가 없는가?
  > ⚠️ **2차 QC에서 수정** — `auth/error/page.tsx`의 `min-vh-60` → `min-h-[60vh]` 수정.
