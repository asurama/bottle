# 6. 최종 테스트 및 외부 QC 체크리스트 (Detailed QC Guide)

'Bottle' 프로젝트의 완성도를 높이기 위한 최종 검수 항목입니다. 이 리스트는 기능적 무결성과 디자인의 프리미엄 감성을 보장하기 위해 작성되었습니다.

---

## 1. 페이지별 기능 검사 (Functional Testing)

### [P1] 홈 페이지 (Home / Listing)
- [ ] **보틀 카드 렌더링**: DB의 실제 데이터(위스키명, 증류소, 가격, 용량)가 정확히 표시되는가?
- [ ] **모집 진행률(Progress)**: 참여 구좌수에 따라 퍼그레시브 바가 실시간(새로고침 시)으로 계산되어 반영되는가?
- [ ] **상태 배지**: 'NEW(미개봉)' 또는 'OPEN(개봉)' 배지가 조건에 맞게 표시되는가?
- [ ] **상세 이동**: 카드 클릭 시 해당 보틀의 상세 페이지(`/share/[id]`)로 정확히 이동하는가?
- [ ] **호버 효과**: 마우스 호버 시 카드가 살짝 떠오르거나 테두리가 강조되는 시각적 피드백이 있는가?

### [P2] 보틀 쉐어 생성 페이지 (Create Share)
- [ ] **폼 유효성 검사**: 필수 항목(이름, 구좌당 가격 등) 누락 시 Zod 에러 메시지가 표시되는가?
- [ ] **데이터 타입**: 숫자가 필요한 필드(ABV, 용량, 가격)에 문자 입력 시 방지 로직이 작동하는가?
- [ ] **보틀 상태 선택**: Select 컴포넌트를 통해 '미개봉/개봉' 상태를 정상적으로 선택할 수 있는가?
- [ ] **상세 설명(Textarea)**: 긴 텍스트 입력 시 레이아웃이 깨지지 않고 정상적으로 저장되는가?
- [ ] **등록 및 리다이렉트**: 등록 완료 후 홈 페이지로 이동하며 최상단에 새 포스트가 노출되는가?

### [P3] 보틀 쉐어 상세 페이지 (Share Detail)
- [ ] **실시간 데이터 로드**: 작성자 정보, 남은 구좌, 이벤트 장소 등의 정보가 정확한가?
- [ ] **로그인 가드**: 로그아웃 상태에서 '로그인하고 참여하기' 버튼과 로그인 페이지 유도가 정상적인가?
- [ ] **참여 버튼(Join)**: 로그인 상태에서 참여 버튼 클릭 시 DB에 `Participation` 레코드가 생성되는가?
- [ ] **참여 중복 방지**: 이미 참여한 사용자의 경우 중복 참여 방지 로직이 작동하는가? (또는 안내 메시지)

---

## 2. 핵심 비즈니스 로직 및 권한 (Business Logic & Auth)

### [L1] 참여 상태 대시보드 (Participation Dashboard)
- [ ] **권한별 버튼 노출**: 
    - [ ] **참여자**: 'PENDING' 상태일 때 '송금 완료(PAID)' 버튼이 보이는가?
    - [ ] **호스트**: 참여자가 'PAID' 상태일 때 '입금 확인(CONFIRM)' 버튼이 보이는가?
- [ ] **상태 변경 반영**: 버튼 클릭 시 서버 액션이 실행되고 페이지가 새로고침(Revalidate)되어 배지가 업데이트되는가?
- [ ] **아이콘 일관성**: 각 상태(대기, 결제, 완료)에 맞는 Lucide 아이콘(Clock, CreditCard, Check)이 표시되는가?

### [L2] 댓글 시스템 (Communication)
- [ ] **댓글 작성**: 텍스트 입력 후 '댓글 게시' 클릭 시 비동기로 저장되고 목록에 즉시 나타나는가?
- [ ] **상대 시간 표시**: '방금 전', 'n분 전' 등 작성 시간이 가독성 있게 표시되는가?
- [ ] **사용자 식별**: 내가 쓴 댓글에 'Me' 배지, 호스트가 쓴 댓글에 'Host' 배지가 붙는가?
- [ ] **빈 댓글 방지**: 공백이나 빈 댓글 입력 시 게시가 차단되는가?

---

## 3. 디자인 일관성 (Design Consistency)

- [ ] **[Amber-Premium] 테마**: 모든 페이지의 배경색(`background`), 주색상(`primary`), 카드 배경(`card`)이 정의된 디자인 시스템을 따르는가?
- [ ] **글라스모피즘 효과**: 카드나 폼 배경에 적용된 `backdrop-blur`와 투명도가 고급스러운 느낌을 주는가?
- [ ] **그라이언트 및 그림자**: 주황색/금색 포인트 부분에 미세한 글로우(Glow) 효과가 적용되어 생동감이 있는가?
- [ ] **반응형 대처**:
    - [ ] 모바일 환경에서 상세 페이지 이미지와 정보 레이아웃이 세로로 유연하게 배치되는가?
    - [ ] 햄버거 메뉴나 네비게이션이 모바일 좁은 화면에서도 원활히 작동하는가?

---

## 4. 최종 배포 전 안점 검사 (Final Safety)

- [ ] **환경변수 보호**: 클라이언트 사이드에서 `DATABASE_URL` 등 민감한 환경변수가 에러 메시지나 콘솔에 찍히지 않는가?
- [ ] **빌드 오류**: `npm run build` 시 TypeScript 타입 오류나 린트 경고가 없는가?
- [ ] **더미 데이터 정리**: 하드코딩된 테스트용 위스키 데이터가 모두 실제 DB 연동 데이터로 교체되었는가?

---
*본 체크리스트는 프로젝트의 최종 완성도를 결정하는 척도입니다. 모든 항목에 체크가 완료되어야 최종 릴리스가 가능합니다.*
