# 3. UI/UX 및 핵심 기능 구현 (Phase 3)

## 1. 왜 이런 작업이 필요한가?
'보틀' 쉐어 플랫폼의 첫인상은 매우 중요합니다. 기획된 MVP 요구사항에 맞춰, **프리미엄 위스키 바와 같은 고급스러운 다크 모드 분위기**를 제공하면서도 글씨는 명확하게 보이도록 화면을 구성해야 합니다. 또한, 주요 핵심 기능인 '보틀 쉐어 목록 조회'와 '게시글 상세 보기 및 댓글 작성'이 직관적이어야 사용자들이 쉽게 참여할 수 있습니다.

## 2. 어떻게 구현하는가?
*   **디자인 시스템**: `Tailwind CSS`와 `shadcn/ui`를 결합하여 다크 모드(`dark` class) 베이스에 Amber/Gold 계열의 Primary 컬러를 적용합니다.
*   **컴포넌트 구조**: 모바일과 패드에서도 예쁘게 보이는 **Card 기반 UI**를 설계하여 메인 홈 화면(보틀 쉐어 목록)을 구성합니다.
*   **상세 페이지 레이아웃**: 보틀의 상세 스펙(이미지, 증류소, 생산년도, 총 쉐어량, 1구좌당 용량 등)을 시각적으로 돋보이게 배치하고, 하단에 댓글 창 및 참여 버튼(입금 확인 절차용)을 둡니다.
*   **더미 데이터 활용**: 백엔드 API 완성을 병행하기 전, UI 스케치를 위해 더미 데이터를 사용해 시각적인 완성도를 먼저 높입니다.

## 3. 작업 목록 및 체크리스트
- [x] 1. 글로벌 색상 테마 업데이트 (Amber/Gold 톤의 카본 다크 테마 `globals.css` 적용)
- [x] 2. `shadcn/ui` 필수 컴포넌트(Card, Button, Input, Badge 등) 추가 설치
- [x] 3. 메인 홈 화면(`app/page.tsx`): 보틀 쉐어 목록을 보여주는 반응형 Grid / Card UI 구현
- [x] 4. 상세 페이지(`app/share/[id]/page.tsx`): 보틀 정보(용량, 가격, 상태) 디스플레이 및 레이아웃 구현
- [x] 5. 댓글 컴포넌트 블록 UI 제작 (아직 실제 DB 통신 전 형태만)

## 4. 작업 결과
*   **Branding**: 위스키 바의 고급스러운 다크 모드와 앰버 골드 테마가 성공적으로 적용되었습니다.
*   **Listing**: 카드 기반의 반응형 목록 UI가 구현되었으며 더미 데이터를 통해 풍성한 시뮬레이션이 가능합니다.
*   **Detail**: 상세 페이지에서 위스키 사양 및 모집 현황을 시각적으로 구현하였고 댓글 레이아웃까지 마쳤습니다.
*   **Verification**: `npm run build` 결과 TypeScript 타입 에러 없이 성공적으로 빌드됨을 확인했습니다.

## 5. 피드백 및 이슈
*   **shadcn conflict**: `shadcn@latest` 설치 시 npm peer dependency 충돌이 발생하여 필수 컴포넌트들을 매뉴얼하게 구성하여 해결했습니다.
*   **Responsive**: 모바일 환경에서도 카드가 겹치지 않도록 `aspect-ratio`를 조절하여 대응했습니다.
