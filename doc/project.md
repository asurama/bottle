Project Name: **Bottle** (보틀 쉐어 웹사이트)

## 1. 개요
비싸거나 희귀한 위스키를 보유하거나 합심하여 돈을 모아 구매하기 위해 보틀을 쉐어하는 플랫폼.
다크 모드와 고급 승인 절차가 들어간 프리미엄 위스키 바 분위기를 제공하는 PC 기반 웹사이트 (우선 MVP)

## 2. 사용자 시나리오 및 기능 기술 (MVP)
*   **회원 시스템**
    *   이메일 기반 가입, 로그인, 로그아웃 (소셜 로그인 x)
    *   가입 후 이메일 승인 절차 (관리자 확인 or 메일 링크) 로직 필요
      - 이메일 verification
      - 관리자 승인
    *   성인 인증 대신 주의 팝업/체크박스로 동의 획득 (대상이 적어 실명인증 생략)

*   **보틀 쉐어 계획(게시글) 기능** 
    *   각 게시글은 보틀 하나를 의미 
    *   보틀 정보 등록 (어떤 위스키인지 등등)
      - 보틀 이미지 업로드
      - 보틀 이름
      - 보틀 정보 (증류소, 생산년도, ADV, years old, etc)
      - 보틀 상태 (신제품, 오픈 (지난기간))
    *   총 쉐어량 (ex. 700ml)
    *   1구좌당 용량 (ex. 100ml)
    *   참여 가능한 전체 구좌 수 산정 (ex. 750ml / 100ml = 7 ) 
    *   참여기간 (ex. 2026-03-04 ~ 2026-03-11)
    *   일정 및 장소 정보 입력  (기본적으로 오프라인 ( 또는 맡겨놓기) )
      - 쉐어일자 (ex. 2026-03-12일 한국도착)
      - 쉐어장소 (아지트, 강남정모 등)

*   **계획 참여 및 결제(정산)**
    *   개인 간 계좌 이체 방식 사용
    *   참여자의 상태 정보 표시 필요: 참여 신청 -> 입금 전 -> 입금 완료 ->(작성자) 입금 확인

*   **커뮤니케이션**
    *   계획 작성자와 소통 가능한 디테일 페이지 내 댓글 기능 (참여자/관심자 작성 가능)

## 3. 기술 스택 제안/선택
*   **Front-end & Meta Framework**: Next.js (App Router), TypeScript
*   **Styling**: Tailwind CSS, shadcn/ui (다크 모드 활용)
*   **Database**: Supabase (PostgreSQL) 
    *   *선정이유*: 관계형 데이터 (회원-게시글-참여자상태-댓글) 관리가 명확하며 넉넉한 무료(Free) 티어를 제공해 개발/MVP 배포에 유리.
*   **ORM**: Prisma
*   **Authentication**: NextAuth.js 활용하여 이메일 credentials 처리 (Magic link or JWT)
*   **Hosting**: Vercel

## 4. UI/UX 디자인 테마
*   **Dark Mode 기본**: 위스키 바의 고급스럽고 어두운, 하지만 글씨는 명확한 앰비언트(Ambient) 느낌의 UI
*   Primary Color는 Amber/Gold 나 차분한 톤 사용 (어두운 배경 + 위스키 컬러)
*   모바일/패드 최적화를 위한 카드형태? 등 제안필요
