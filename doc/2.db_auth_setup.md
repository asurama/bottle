# 2. 데이터베이스 및 인증 시스템(Auth) 연동

## 1. 왜 이런 작업이 필요한가?
'보틀' 쉐어 플랫폼은 사용자를 구별하고, 데이터 간의 관계(Join) 설정 및 트랜잭션 관리에 최적화된 PostgreSQL 환경을 마련하고 인증 절차(NextAuth)를 연결해야 합니다. (무료 서버인 Supabase 사용)

## 2. 어떻게 구현하는가?
*   Prisma ORM을 설치하고 데이터베이스 스키마(`User`, `BottleShare` 등)를 정의하여 Supabase에 반영(push)합니다.
*   `NextAuth.js` 라이브러리를 통해 로그인/이메일 인증 기반의 자격 증명(Session)을 관리할 수 있도록 `api/auth/[...nextauth]/route.ts` 인프라를 마련합니다.
*   Nodemailer 기반이나 Resend API 같은 메일 프로바이더를 연동해 "이메일 승인(확인)" 파이프라인을 구축합니다.

## 3. 작업 목록 및 체크리스트
- [x] 1. Prisma 클라이언트 설치 및 `.env` 파일에 Supabase 접속 URI 세팅
- [x] 2. Prisma Schema 초기 구조(`User` 등 핵심 테이블) 작성
- [x] 3. `NextAuth.js` 패키지 설치 및 Prisma Adapter 연결 
- [x] 4. 이메일 기반 인증 API 구성 (Magic Link 방식 우선 검토)
- [x] 5. Supabase 프로젝트 접속 설정 확인 및 푸시

## 4. 작업 결과
*   Prisma + Supabase 연동을 위한 스키마(`User`, `BottleShare`, `Participation`, `Comment` 등) 세팅 완료.
*   NextAuth의 `v5` (Auth.js)를 사용하여 `src/auth.ts` 와 `src/app/api/auth/[...nextauth]/route.ts` 기본 인프라 생성 완료.
*   이메일 매직 링크를 위한 `Nodemailer` 연동 기초 환경 구축.

## 5. 피드백 및 이슈
*   **[완료]** 사용자님의 Supabase 접속 URL 및 패스워드를 `.env`에 성공적으로 반영.
*   **[완료]** IPv4 환경 제약을 고려하여 Supabase `DIRECT_URL` (5432 포트 풀러)를 활용한 `npx prisma db push` 명령어를 완료하였으며 테이블(`User`, `BottleShare`, `Participation`, `Comment` 등)이 올바르게 생성됨을 확인했습니다.
