# 4. 비즈니스 로직 및 상태 관리 (Phase 4)

## 1. 왜 이런 작업이 필요한가? (Rationale)
단순한 게시판을 넘어, **실제 '쉐어 참여'와 '입금 확인' 과정**이 시스템화되어야 유저 간의 신뢰를 형성할 수 있습니다. 수동으로 계좌 이체를 확인하더라도, 시스템상에서 '입금 전', '입금 완료', '확인됨' 등의 상태를 명확히 보여줌으로써 분쟁을 예방하고 진행 상황을 투명하게 공개합니다.

## 2. 어떻게 구현하는가? (Implementation Design)
*   **Server Actions**: 데이터 변조 방지를 위해 Next.js Server Actions를 사용하여 게시글 생성 및 참여 상태 변경을 처리합니다.
*   **Validation**: `Zod`를 활용하여 입력값(용량, 가격, 날짜 등)의 유효성을 서버 측에서 엄격하게 검증합니다.
*   **상태 머신 (Status Machine)**: 
    - `SharePost`: `RECRUITING` -> `COMPLETED` -> `CANCELLED`
    - `Participation`: `PENDING` (신청) -> `PAYMENT_DONE` (입금완료 알림) -> `CONFIRMED` (주최자 확인)
